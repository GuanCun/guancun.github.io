CPU 过载与温度监控：我的 MacBook 使用体验

2024-10-30T17:18:53+08:00

今天偶然发现我的 MacBook 变得非常烫手，这是后台过量进程消耗系统资源的表现，而我当前并未运行大量内存或高功耗的应用程序。 通过「活动监视器」，我追查到 VSCode 的 Code Helper 进程，它占用单个 CPU 竟然达到惊人的 100%。并且存在多个类似的进程。 ![[Pasted image 20241030155055.png]] 由于我通常使用外接设备而非直接接触到 MacBook，这让我对笔记本的温度异常难以察觉，对系统运行情况一无所知。 这件事促使我开始关注系统的监控，通过 perplexity 搜索，我找到两款推荐的监控工具： State：它能够实时在你的菜单栏上显示Mac电脑的CPU、内存、硬盘、温度、风扇、网络等信息。 RunCat：一个通过有趣的 GIF 动画在菜单栏显示 CPU 负载的应用。 两款 APP 的使用体验对我来说差异不大，但是我还是选了更抽象的 RunCat，其直观简单的逻辑设计更符合我当前的需求。 ![[Pasted image 20241030160538.png]]
